## Description

This diagram represents the deployment's architecture of the system:
- **User Interface:** Users interact with the system primarily through Discord via slash commands. A dedicated Streamlit dashboard is planned (V3.0) to provide advanced data visualization through a web interface.
- **Application Hosting (Oracle Cloud):** The core application (Discord Bot) is planned to be hosted on an Oracle Cloud VM. We plan to utilize Docker Containers to ensure environment isolation and scalability.
- **Data Persistence:** The system uses Neon Tech (PostgreSQL) as a centralized cloud database, ensuring data consistency across all components.
- **Automation (Github Actions):** Github Actions serves two critical roles:
    - **Data Acquisition:** It executes the web scraper on a scheduled basis (Cron Job), persisting the extracted data directly into Neon Tech.
    - **CI/CD Pipeline:** Planned to automate the build proess, creating docker images upon code update and pushing them to Docker Hub for deployment.
