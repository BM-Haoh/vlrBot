## Description
Layer based system:

**1. Models:**
- Data structure and entities (Match, Map, Pickban) that represent the Valorant competitive domain.

**2. Utils:**
- Helper functions providing reusable logic for web automation (Selenium) and data manipulation (Pandas).

**3. Controller:**
- The system's entry point that orchestrates the execution flow between scraping and data persistence.

**4. Services:**
- The core business logic of the application. Contains:
    - vlr_stealer: scrapper responsible for navigating and extracting match data from vlr.gg.
    - stats_manager: Manages the lifecycle of statistics, coordinating data cleaning via Pandas and triggering storage.
    - DB_handler: Does the extensive database operations, managing connections and SQL execution related to matches for the Neon Tech (PostgreSQL) instance.

## Description
layer based system:

**1. Models:**
- Data structure and entity (cache) that represent the Valorant competitive domain.

**2. Utils:**
- Helper functions providing reusable logic for asynchronous database loading (SQL Selects) and UI components for Discord (pagination).

**3. Services:**
- The core business logic of the application. Contains:
    - Brain: main logic, all the calculations and data analysis transformations.
    - Discord_Bot: System controller that maps Discord slash commands to the Brain's logic and manages bot's lifecycle and events.
